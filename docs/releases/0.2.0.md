# rusty-alto 0.2.0

Version 0.2.0 expands rusty-alto from its initial string/tree IRTG foundation
into a broader Alto-compatible grammar toolkit, with TAG support, application
APIs for graphical frontends, and typed codec discovery.

## Highlights

### TAG and feature structures

- Added Alto-compatible TAG string and TAG derived-tree algebras, including
  decomposition automata and binarized TAG-tree support.
- Added immutable feature structures with reentrancy, parsing, unification,
  projection, embedding, and multi-attribute remapping.
- Added feature-structure filtering for rejecting derivations whose feature
  terms fail to evaluate.
- Added `TulipacInputCodec`, which reads `.tag` grammars and compiles them into
  IRTGs with string, tree, and optional feature-structure interpretations.
- Tulipac path loading supports relative `#include` directives.

### Unified codec infrastructure

- `InputCodec<T>` is now object-safe and exposes shared `CodecMetadata`.
- Added `InputCodecRegistry`, with exact-result-type lookup by codec name,
  filename extension, or path.
- The standard input registry maps `.irtg` and `.tag` to `Irtg`, and `.auto`
  to `ExplicitWithSignature`.
- Added a common `InputCodecError` that retains concrete parse and I/O errors
  as error sources.
- `OutputCodec<V>` now has an associated output type and shared metadata.
- Added `OutputCodecRegistry` for textual encodings of exact public algebra
  value types.
- Added algebra-owned display codecs and `VisualRepresentation` for text,
  trees, and feature structures. Metadata enumeration remains lazy and does
  not evaluate or encode values.

### Frontend-facing APIs

- Added summaries, resolved rules, interpretation metadata, language
  cardinality, and frontend-friendly parse strategies.
- Added structured derivation rendering that preserves tree values.
- Parse charts now retain human-readable state labels where available.
- The interactive `rusty-alto` frontend now opens both `.irtg` and `.tag`
  grammars through the input-codec registry.

### Parsing and performance

- Extended parse-input support to TAG string and TAG tree interpretations.
- Added reusable prepared indexes for repeated string A* parsing.
- Improved A* candidate generation and fallback handling for noncanonical
  unary, binary, and higher-arity homomorphic images.
- Added product-state identity reporting to condensed materializers.
- Continued the project-wide emphasis on lazy indexes, dense IDs, and
  allocation-conscious common paths.

## Breaking API changes

- `InputCodec<T>` no longer has a codec-specific associated error or a generic
  `read<R: Read>` method. Implementations now provide object-safe stream
  reading and return `InputCodecError`.
- `OutputCodec<V>` now has an associated `Output` type and a `metadata`
  method.
- `ParsedTreeAutomaton` was renamed to `ExplicitWithSignature`.
  `ParsedTreeAutomaton` remains as a deprecated type alias for migration.
- Implementors of `Algebra` must provide `visualize(&Self::Value)`.

## Migration notes

Load a grammar according to its extension:

```rust
use rusty_alto::{InputCodecRegistry, Irtg};

let registry = InputCodecRegistry::standard();
let path = std::path::Path::new("grammar.tag");
let grammar = registry.codec_for_path::<Irtg>(path)?.read_path(path)?;
# Ok::<(), Box<dyn std::error::Error>>(())
```

Discover textual representations of an algebra value:

```rust
use rusty_alto::{OutputCodecRegistry, TreeValue};

let registry = OutputCodecRegistry::standard();
for codec in registry.codecs_for::<TreeValue>() {
    println!("{}", codec.metadata().description);
}
```

See [`../codec-infrastructure.md`](../codec-infrastructure.md) for the codec
model and a concrete GUI adaptation checklist.
